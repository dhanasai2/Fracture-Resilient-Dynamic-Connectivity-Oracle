💡 How ShatterMap Works

1

Build & Classify

Load all edges, run DFS to identify bridge edges (stress) vs non-bridge edges (reinforced). In sparse backbone networks, <1% are bridges.

2

O(1) Safe Deletes

When deleting a reinforced edge, connectivity is GUARANTEED to be preserved. Just remove it — no verification needed. This is the key innovation.

3

Fracture Response

When deleting a stress edge (bridge), trigger BFS to detect component split and reclassify edges. Rare event (<1% of operations).

4

O(1) Queries

Every node has a zone ID. Connected nodes share the same zone. Connectivity check = compare two integers.
